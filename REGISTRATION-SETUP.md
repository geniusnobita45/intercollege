# Online registration setup

## What is included

The `/registration` route contains a responsive four-step form covering:

1. Student identity and date of birth
2. Parent or guardian contact and address
3. Applying class and previous-school information
4. Documents, medical/support notes and declarations

The form autosaves text fields in the browser, validates each step, accepts common image/PDF documents, generates an application reference and provides a printable confirmation.

## Netlify Forms activation

1. Run `npm run build`.
2. Deploy the generated `dist` folder to Netlify, or connect the repository to Netlify.
3. Submit one test application on the deployed domain.
4. Open Netlify → Forms and confirm that `student-registration` appears.
5. Configure email notifications for the admission office.
6. Review file-upload limits and retention settings for the selected Netlify plan.

The static form in `index.html` allows Netlify to detect fields generated by React.

## Before public launch

- Confirm the classes and streams currently open for admission in `src/data/college.ts`.
- Add the college’s admission rules, fee information and document checklist.
- Publish a privacy notice covering children’s and parent/guardian data.
- Decide who may access submissions and how long records are retained.
- Test the form on the final domain with a real notification recipient.
- Do not treat online registration as confirmed admission; retain document and seat verification.

## Other hosting providers

Netlify Forms is the default no-backend submission path. For Vercel or another host, replace the `fetch('/')` call in `src/pages/RegistrationPage.tsx` with a secure API endpoint or approved form service. Do not send sensitive application data through a public client-side email service.
